# Probability for Deep Learning

Deep learning does not only learn deterministic prediction functions. Many modern models learn or manipulate **probability distributions**:

- classifiers learn $p_\theta(y \mid x)$
- language models learn $p_\theta(x_t \mid x_{<t})$
- VAEs learn latent distributions
- diffusion models learn denoising processes related to scores and probability densities
- calibration asks whether model probabilities can be trusted

So probability is a broad foundation: it sits underneath loss functions, generative models, representation learning, uncertainty, sampling, and evaluation.

<Note type="intuition">
If linear algebra is the language of vector representations, probability is the language of **uncertainty**. It asks: what does the model believe, what distribution does the data follow, and how far is the model from that data distribution?
</Note>

## Where probability appears in deep learning

Probability concepts in deep learning rarely appear as isolated definitions. They usually come attached to concrete questions: which distribution is a loss averaged over, how much probability does the model assign to real data, how is uncertainty measured, or how does sampling produce new data?

## A small map

| Concept | Question |
|---|---|
| Random variable | What quantity is uncertain? |
| Distribution | How is probability spread across values? |
| Expectation | What is the long-run average of a quantity? |
| Likelihood | How much probability does the model assign to observed data? |
| Entropy | How uncertain is the true distribution? |
| Cross-entropy | How costly is it to encode true data using the model? |
| KL divergence | How far is the model from the true distribution? |
| Sampling | How do we draw samples from a distribution? |

The key point: these concepts do not live separately. In supervised classification, cross-entropy is negative log-likelihood with one-hot labels. In variational inference, KL measures the mismatch between an approximate posterior and a prior or true posterior. In language modeling, perplexity is a more readable scale for cross-entropy.

## The likelihood, cross-entropy, and KL triangle

Likelihood, cross-entropy, and KL divergence are the three most common lenses for reading loss functions in deep learning:

1. **Likelihood** reads from the model's side: does the model assign high probability to the observed data?
2. **Cross-entropy** reads from the coding/loss side: how costly is it to encode true data using the model?
3. **KL divergence** reads from the mismatch side: how different is the model distribution from the true distribution?

The same training objective can be read through all three lenses. Once that connection is clear, cross-entropy loss is no longer just "the formula used in classifiers"; it becomes a way to measure mismatch between the model distribution and the data distribution.
