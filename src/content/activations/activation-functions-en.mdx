## Why do we need activation functions?

Without activation functions, an $L$-layer neural network is just a linear transformation:

$$f(x) = W_L \cdots W_2 W_1 x = W_{\text{eff}} x$$

The entire network collapses into a single matrix multiplication — incapable of representing non-linear relationships. Activation functions break this linearity after each layer:

$$h^{(l)} = \sigma(W^{(l)} h^{(l-1)} + b^{(l)})$$

**Universal Approximation Theorem** (Hornik 1991): a single-hidden-layer network with a non-linear activation and sufficient neurons can approximate any continuous function on a compact set. But "sufficient neurons" can be astronomically large — that's why we need depth, and depth requires activations that don't cause vanishing gradients.

---

## Three core problems to solve

### 1. Vanishing Gradient

Backpropagation propagates gradients backwards through the chain rule:

$$\frac{\partial \mathcal{L}}{\partial W^{(1)}} = \frac{\partial \mathcal{L}}{\partial h^{(L)}} \cdot \prod_{l=2}^{L} \frac{\partial h^{(l)}}{\partial h^{(l-1)}}$$

Each factor in the product is $\sigma'(z^{(l)}) \cdot W^{(l)}$. If $|\sigma'| < 1$ — like Sigmoid with $\sigma'_{\max} = 0.25$ — the gradient is multiplied by a number less than 1 at every layer:

$$\text{10-layer Sigmoid network}: \quad (0.25)^{10} \approx 10^{-6}$$

The gradient at the first layer is effectively zero → early layers learn nothing.

<Note type="theorem" label="Necessary condition">
To prevent vanishing gradients, we need $|\sigma'(x)| \approx 1$ in active regions, or $\sigma'(x) \neq 0$ for all $x$ where the neuron fires.
</Note>

### 2. Zero-Centering

If an activation always outputs positive values (like Sigmoid: output $\in (0,1)$), then all gradients with respect to weights in the next layer share the same sign:

$$\frac{\partial \mathcal{L}}{\partial w_{ij}} = \frac{\partial \mathcal{L}}{\partial z_j} \cdot h_i, \quad h_i > 0$$

The sign of $\partial \mathcal{L}/\partial w_{ij}$ depends only on $\partial \mathcal{L}/\partial z_j$ — all weights in the same layer must increase or decrease together. This forces gradient descent into a zig-zag path through weight space instead of heading directly toward the minimum.

<Note type="intuition">
Zero-centered activations (mean output $\approx 0$) allow weights in the same layer to update in different directions → faster convergence.
</Note>

### 3. Dying Neurons (Dead Neuron Problem)

With ReLU: if $z = Wx + b < 0$ for all inputs in the training set, the gradient at that neuron is permanently zero:

$$\text{ReLU}'(z) = 0 \quad \forall z < 0 \implies \frac{\partial \mathcal{L}}{\partial W} = 0$$

The neuron "dies" — it never recovers. This typically happens with a learning rate that's too high or poor weight initialization. In some models, 10–40% of ReLU neurons can be dead.

## Early era: Sigmoid and Tanh (1986–2009)

### Sigmoid — $\sigma(x) = \frac{1}{1+e^{-x}}$

Derived from logistic regression. Gradient:

$$\sigma'(x) = \sigma(x)(1 - \sigma(x))$$

Maximum at $x=0$: $\sigma'(0) = 0.25$. Output $\in (0,1)$ — not zero-centered.

Still used for: gates in LSTM/GRU (requires output $\in (0,1)$ to act as a "valve"), binary classification output layers.

### Tanh — $\tanh(x) = \frac{e^x - e^{-x}}{e^x + e^{-x}}$

Essentially a rescaled Sigmoid: $\tanh(x) = 2\sigma(2x) - 1$.

$$\tanh'(x) = 1 - \tanh^2(x)$$

Maximum gradient = 1 at $x=0$. Zero-centered (output $\in (-1,1)$) → faster convergence than Sigmoid. But still saturates at large $|x|$ → vanishing gradient not fully solved.

## The breakthrough: ReLU (2010)

$$\text{ReLU}(x) = \max(0, x), \quad \text{ReLU}'(x) = \mathbf{1}[x > 0]$$

Krizhevsky et al. (2012) used ReLU in AlexNet and won ImageNet by a large margin — launching the deep learning era. The theoretical foundation came from Vinod Nair & Geoffrey Hinton (2010).

**Why ReLU works:**

- Gradient = 1 for $x > 0$: no vanishing in the positive direction
- Sparse activation: ~50% of neurons inactive on average → computationally efficient, implicit regularization
- Extremely fast to compute: just `max(0, x)`

<Note type="result">
Hinton et al. reported ReLU converges approximately **6× faster** than Sigmoid on deep networks.
</Note>

**The math behind sparsity:** ReLU produces sparse representations — only a small subset of neurons are active for each input. This is analogous to sparse coding in neuroscience and acts as implicit L1 regularization on activations.

## The ReLU family: fixing dying neurons

### Leaky ReLU & PReLU

$$\text{Leaky ReLU}(x) = \begin{cases} x & x > 0 \\ \alpha x & x \leq 0 \end{cases}, \quad \alpha = 0.01$$

A small slope in the negative region keeps gradients non-zero. **PReLU** (He et al., 2015) makes $\alpha$ a learnable parameter — each neuron (or channel) learns its own $\alpha$. He et al. used PReLU to surpass human-level accuracy on ImageNet for the first time.

### ELU — Exponential Linear Unit

$$\text{ELU}(x) = \begin{cases} x & x \geq 0 \\ \alpha(e^x - 1) & x < 0 \end{cases}$$

Smooth at $x=0$ (unlike the corner in ReLU/Leaky ReLU). Mean activation closer to 0 than ReLU because the negative region pulls it down. Downside: requires computing $e^x$ → slower.

### SELU — Self-Normalizing ELU

$$\text{SELU}(x) = \lambda \cdot \text{ELU}(x, \alpha)$$

$$\lambda = 1.0507, \quad \alpha = 1.6733$$

These are not arbitrary constants. Klambauer et al. (2017) derived them by solving a **fixed-point problem**: if the input has distribution $\mathcal{N}(0,1)$, find $\alpha$ and $\lambda$ such that the output also has mean $= 0$, variance $= 1$.

<Note type="theorem" label="Self-normalization property">
With the $\alpha, \lambda$ above and lecun_normal weight initialization $\mathcal{N}(0, 1/n)$, each SELU layer acts as a contraction mapping that pulls the distribution toward $\mathcal{N}(0,1)$. No BatchNorm needed.
</Note>

Limitations: only works with lecun_normal init, incompatible with standard dropout (requires AlphaDropout to preserve normalization), and rarely used in practice despite elegant theory.

## Neural Architecture Search: Swish (2017)

$$\text{Swish}(x) = x \cdot \sigma(x) = \frac{x}{1 + e^{-x}}$$

Ramachandran et al. (Google Brain) used **Neural Architecture Search** — automatically searching over the space of mathematical functions — and Swish emerged as the best-performing activation across multiple benchmarks.

$$\text{Swish}'(x) = \sigma(x) + x \cdot \sigma(x)(1-\sigma(x)) = \text{Swish}(x) + \sigma(x)(1 - \text{Swish}(x))$$

Two distinctive properties:

1. **Non-monotonic**: has a local minimum around $x \approx -1.28$. Unlike ReLU and Sigmoid which are both monotone. Non-monotonicity lets the network represent more complex functions.

2. **Smooth**: continuously differentiable everywhere — smoother optimization landscape than ReLU's sharp corner.

SiLU (Sigmoid Linear Unit) is an alternative name for Swish — same formula, named independently by Elfwang et al. 2017.

## Transformer era: GELU (2016/2018)

$$\text{GELU}(x) = x \cdot \Phi(x)$$

Where $\Phi(x) = P(X \leq x)$ with $X \sim \mathcal{N}(0,1)$ is the standard normal CDF.

**Probabilistic interpretation:** GELU weights the input $x$ by the probability that a random Gaussian sample is less than $x$. When $x$ is large and positive, $\Phi(x) \approx 1$ → output $\approx x$ (like ReLU). When $x$ is large and negative, $\Phi(x) \approx 0$ → output $\approx 0$.

Connection to **stochastic regularization**: GELU can be viewed as the expectation of a stochastic gate:

$$\text{GELU}(x) = \mathbb{E}[\text{Bernoulli}(\Phi(x)) \cdot x]$$

Each neuron is dropped with probability $1 - \Phi(x)$, proportional to how negative the activation is. GELU is a deterministic approximation of this process.

<Note type="intuition">
GELU = stochastic ReLU. Instead of a hard threshold at 0, GELU uses a soft, probabilistic threshold. This smooths the loss landscape and aids optimization.
</Note>

Practical approximation (used in implementations):

$$\text{GELU}(x) \approx 0.5x\left(1 + \tanh\!\left[\sqrt{\tfrac{2}{\pi}}\left(x + 0.044715x^3\right)\right]\right)$$

BERT, GPT-2, GPT-3, ViT, and most Transformers after 2018 use GELU.

## LLM era: GLU and SwiGLU (2020–present)

### Gated Linear Unit — GLU

$$\text{GLU}(x, W, V) = \sigma(xW) \odot xV$$

This is a **gating mechanism**: $\sigma(xW)$ acts as a gate — selectively filtering information from $xV$. The gate learns "which features should pass through" depending on the input.

### SwiGLU

$$\text{SwiGLU}(x, W, V) = \text{Swish}(xW) \odot xV$$

Replaces the Sigmoid gate with Swish — smoother, better gradients.

**In the Transformer FFN**, SwiGLU replaces ReLU/GELU as follows:

| Standard FFN | SwiGLU FFN |
|---|---|
| $\text{FFN}(x) = \max(0, xW_1)W_2$ | $\text{FFN}(x) = (\text{Swish}(xW_1) \odot xW_3)W_2$ |
| 2 matrices: $W_1, W_2$ | 3 matrices: $W_1, W_2, W_3$ |

SwiGLU needs 3 projections instead of 2 → to maintain the same total parameter count, the hidden dimension is reduced from $4d$ to $\frac{8d}{3}$.

<Note type="result">
Noam Shazeer (2020) tested many GLU variants and SwiGLU gave the best results on language modeling perplexity. LLaMA, PaLM, Mistral, and Gemma all use SwiGLU in their FFN layers.
</Note>

## Summary: which activation to use

| Use case | Activation |
|---|---|
| CNN, MLP (default) | ReLU |
| Dead neuron problems | Leaky ReLU or ELU |
| No BatchNorm desired | SELU (with lecun_normal init) |
| Transformer encoder/decoder | GELU |
| LLM / Large Language Model | SwiGLU |
| Binary classification output | Sigmoid |
| Multiclass output | Softmax |
| LSTM/GRU gates | Sigmoid (forget/input/output) + Tanh (cell state) |
| Vision SOTA (EfficientNet...) | Swish / SiLU |

**General rules:**
- Start with ReLU — fast, simple, works well on most tasks
- Switch to GELU for Transformer architectures
- Use SwiGLU when training LLMs from scratch
- Never use Sigmoid/Tanh in hidden layers of deep networks
