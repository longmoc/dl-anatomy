# State Space Models (Mamba)

The [Sequence Models](/dl-anatomy/architectures/sequence-models-en) article ended on a dilemma: RNNs keep a compact state but are **sequential** (no parallelism); Transformers parallelize fully but cost $O(n^2)$ with an ever-growing KV cache. **State Space Models** try to take the best of both.

## The idea: a linear system from control theory

SSMs borrow a classic model from control theory — a system with a continuous hidden state that, once discretized, gives:

$$
h_t = A h_{t-1} + B x_t, \qquad y_t = C h_t
$$

It looks just like an RNN, but with one crucial difference: the update is **linear** (no $\tanh$ or nonlinearity wedged between steps).

## Why linearity matters

Because the recurrence is linear, we can *unroll* it into a **convolution** with a precomputed kernel:

$$
y = x * \bar{K}, \qquad \bar{K} = (C B,\; C A B,\; C A^2 B,\; \dots)
$$

As a result an SSM has **two equivalent modes**:

| Mode | Used when | Cost |
|---|---|---|
| Convolution (parallel) | Training | $O(n)$, parallel over the whole sequence |
| Recurrence (sequential) | Inference | Fixed state, no ever-growing KV cache |

This is something the Transformer lacks: train in parallel like a Transformer, infer compactly like an RNN.

<Note type="intuition">
Attention remembers *every* past token (KV cache grows with length). An SSM compresses the past into a fixed-size state — like an RNN. The trade-off: a fixed state can miss distant detail, so the design of $A$ (how the state "forgets") is the crux.
</Note>

## Mamba — selective state

Classic SSMs have **fixed** $A, B, C$ — they treat every token the same and can't filter information by content (their weakness vs attention). **Mamba (Gu & Dao, 2023)** makes $B, C$ (and the discretization step) **input-dependent**:

$$
B = B(x_t), \quad C = C(x_t)
$$

→ the model decides what to *keep or forget* per token — the **selective scan**. This is the piece that lets Mamba close the quality gap with attention while staying $O(n)$ in length.

The cost: input-dependent updates break the fixed-convolution form, so Mamba uses a hand-optimized **parallel scan** on the GPU to stay parallelizable during training.

## The big picture

$$
\text{RNN} \;\to\; \text{LSTM/GRU} \;\to\; \text{Transformer} \;\to\; \text{SSM/Mamba}
$$

- RNN: sequential, vanishing
- LSTM/GRU: gating for memory, still sequential
- Transformer: parallel, great long-range, but $O(n^2)$
- SSM/Mamba: linear $O(n)$, selective, compact inference

<Note type="result">
Open question of 2024+: can SSMs replace attention for **very long** sequences (genomics, audio, million-token context)? In practice many hybrid models interleave attention and SSM layers — getting attention's precise long-range mixing in a few layers, and SSM's linear cost in the rest.
</Note>
