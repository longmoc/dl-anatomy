# Transformer & Attention

In 2017, *"Attention Is All You Need"* dropped recurrence entirely: no RNN, no convolution — just **attention**. It's the foundation of nearly every modern LLM.

## Scaled dot-product attention

The idea: each token produces three vectors — a **query** $q$ (what am I looking for), a **key** $k$ (what do I contain), a **value** $v$ (what information do I carry). Token $i$ aggregates the values of other tokens, weighted by how well its query matches their keys:

$$
\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right) V
$$

- $QK^\top$ — match scores between every pair of tokens
- dividing by $\sqrt{d_k}$ — normalization: when $d_k$ is large, the dot product has high variance → softmax saturates (near-zero gradients); dividing by $\sqrt{d_k}$ keeps the variance stable
- softmax — turns scores into weights that sum to 1

<Note type="intuition">
Attention is a "soft dictionary lookup": the query is compared against every key, and values are retrieved weighted by match. Unlike a hard hashtable (one key hits), here every key contributes a share.
</Note>

## Self-attention is permutation-equivariant

In **self-attention**, $Q, K, V$ all come from the same sequence → each token attends to every token (including itself). An important consequence: attention is **permutation-equivariant** — shuffle the tokens and the output just shuffles too; attention doesn't "know" position. Fine for sets, but for text order is everything.

## Positional encoding — injecting position

Since attention has no notion of order, we must add position to the input:

- **Sinusoidal PE (2017)**: add multi-frequency sine/cosine waves to the embedding — absolute position, fixed, no learning needed
- **Learned PE**: learn a vector per position (BERT, GPT-2)
- **RoPE (2021)**: *rotate* the q/k vectors by an angle proportional to position → encodes **relative** position, extrapolates better to long sequences; used in LLaMA and most recent LLMs
- **ALiBi (2021)**: add a distance-linear bias to the attention scores

## Multi-head attention

Instead of one attention, use $h$ parallel "heads", each learning a different kind of relation (syntax, coreference, …), then concatenate. Like multiple kernels in a CNN — several relational filters running at once.

## The Transformer block

A block stacks: **multi-head attention → FFN** (a 2-layer network applied per token), each wrapped in a **residual connection** (see [CNN & ResNet](/dl-anatomy/architectures/cnn-resnet-en)) + **normalization**. Stack $N$ blocks → a Transformer.

Why it beats RNNs: (1) **parallelizable** over the whole sequence during training, (2) the path between any two tokens is just 1 step → long-range dependencies are easy to capture, (3) it **scales** smoothly with data and compute.

## Variants

| Variant | Example | Used for |
|---|---|---|
| Encoder-only | BERT | Understanding/representation (classification, embeddings) |
| Decoder-only | GPT, LLaMA | Autoregressive text generation |
| Encoder-decoder | T5, BART | Seq2seq (translation, summarization) |
| Vision Transformer | ViT | Images split into patches → tokens |
| Efficient Transformer | Longformer, FlashAttention | Reducing the $O(n^2)$ cost |

## MoE — scaling parameters without scaling compute

Attention is $O(n^2)$ expensive, but most LLM parameters live in the FFN. **Mixture of Experts** replaces one FFN with many "expert" FFNs + a **router**: each token passes through only the top-$k$ experts (often $k=2$).

$$
y = \sum_{e \in \text{top-}k} g_e(x)\, \text{FFN}_e(x)
$$

→ total parameters multiply (more knowledge) but **per-token compute stays roughly constant** (sparse activation). Switch Transformer and Mixtral use this. The challenge: **load balancing** — preventing the router from collapsing onto a few experts.

<Note type="result">
A Transformer is a GNN on a complete graph (every token connected, edge weights = attention). It drops the inductive biases of order/locality, requiring large data + compute in return — and that very "assumption-free" nature is what lets it generalize to text, images, audio, proteins, and more.
</Note>
