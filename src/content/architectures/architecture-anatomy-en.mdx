# Architecture Anatomy

An **architecture** is the structural design of a model — how layers connect, how information flows, and how the model is organized to solve a class of problems. It's distinct from the *optimizer* (how you train) and the *activation* (the per-neuron nonlinearity). The central question of any architecture:

> What **inductive bias** does this design encode — which assumptions about data structure are baked into the model?

CNNs assume *locality* (nearby pixels matter). RNNs assume *order* (sequential signals). GNNs assume *relations* (a graph). Transformers assume *almost nothing* about structure — they learn relations from data, at the cost of needing more data and compute.

This series moves through two layers:

1. **Backbone architectures** — how information moves through the model: convolution, recurrence, attention, message passing.
2. **Model families** — how those backbones are organized into larger modeling patterns: representation learning, generative modeling, or multimodal alignment.

<Note type="intuition">
Backbone architectures ask "what mechanism moves information through the model?". Model families ask "how do we use those mechanisms to learn latent spaces, generate data, or align multiple kinds of signals?".
</Note>

## One shared question: where does information come from?

One useful way to read architectures is to ask: at each layer, which other units is a unit allowed to take information from?

In a CNN, a pixel or patch mainly sees a local neighborhood. In an RNN, the current timestep sees the previous state. In a GNN, a node sees the nodes connected to it by edges. In a Transformer, each token can see every other token through attention.

In more general language, this is **message passing**: each unit receives "messages" from a set of information sources, aggregates them, and updates its representation. Architectures differ mostly in how they define that source set.

| Architecture | Data structure | Who does each unit look at? |
|---|---|---|
| CNN | Image grid | Nearby pixels/patches in the kernel |
| RNN | Sequence | The previous step's state |
| GNN | Arbitrary graph | Nodes connected by edges |
| Transformer | Sequence as a complete graph | *Every* other token through attention |

<Note type="intuition">
Through the message passing lens, a Transformer can be seen as a **GNN on a complete graph**: every token connects to every other token, and the edge weights are the attention scores. This is why it is so flexible but also expensive — if every token talks to every other token, the number of interactions grows as $O(n^2)$.
</Note>

## The evolution axis

The mainstream of modern architectures trends toward *fewer hand-designed assumptions, more scale, and letting data do the learning*:

| Era | Architecture | Leap |
|---|---|---|
| ~2012 | CNN (AlexNet) | Deep learning truly takes off on ImageNet |
| 2015 | ResNet | Residual connections → trainable 1000+ layer nets |
| 2017 | Transformer | Self-attention, no recurrence, parallelizable |
| 2020+ | ViT, MoE, SSM (Mamba) | Scaling parameters, beyond attention |
