# Sequence Models: RNN → Seq2Seq

The [Modern CNNs](/dl-anatomy/architectures/modern-cnns-en) article ended in the world of **image grids**: locality, weight sharing, hierarchy. Sequential data — text, audio, time-series — has a different structure: **order**.

In an image, a pixel has a fairly stable spatial neighborhood. In a sequence, the current element depends on what came before, sometimes very far back. The natural inductive bias for sequence modeling is therefore:

> process step by step, carry a **state** that summarizes the past, and use that state to interpret the present.

The evolution in this article starts from the idea of state: RNNs compress the past into a hidden state; LSTM/GRU make that state remember longer with gates; Seq2Seq uses state to map one sequence into another; attention breaks the bottleneck by turning all hidden states into a memory bank. The Transformer then keeps attention and drops recurrence. SSM/Mamba returns to state, but organizes it to scale linearly.

## RNN — state as compressed memory

A Recurrent Neural Network maintains a hidden state $h_t$, updated each step from input $x_t$ and the previous state:

$$
h_t = \tanh(W_h h_{t-1} + W_x x_t + b)
$$

The same weights are reused at every step — **weight sharing across time**. If we unroll the RNN, we get a very deep network along the time axis: the same cell repeated from $t=1$ to $t=T$.

<RNNUnrolledDiagram lang="en" />

The hidden state $h_t$ is a compressed summary of the past:

$$
h_t \approx \text{summary}(x_1, x_2, \dots, x_t)
$$

The strength: an RNN handles variable-length sequences because the same cell can run for as many steps as needed. The weakness: all past information must pass through a fixed-size vector. As the sequence grows, the state must both preserve old information and absorb new information — a natural **information bottleneck**.

## Temporal Depth and Vanishing Gradients

The deeper problem appears in backpropagation through time. A gradient from the last step back to an early step must pass through a chain of Jacobians:

$$
\frac{\partial L}{\partial h_t}
=
\frac{\partial L}{\partial h_T}
\prod_{k=t+1}^{T}
\frac{\partial h_k}{\partial h_{k-1}}
$$

For a tanh RNN, each Jacobian contains both the recurrent matrix $W_h$ and the derivative of the activation:

$$
\frac{\partial h_k}{\partial h_{k-1}}
=
\operatorname{diag}(1 - h_k^2)\, W_h
$$

Multiplying many Jacobians makes gradients **vanish** if the effective singular values are below 1, or **explode** if they are above 1. This is why vanilla RNNs struggle with dependencies beyond a few dozen steps.

<Note type="warning">
The problem is not just that an RNN "forgets" in a psychological sense. Mathematically, time creates a very deep network along the temporal dimension; gradients must cross that depth, so RNNs face a difficulty similar to very deep networks before ResNet.
</Note>

## LSTM/GRU — Gating to Preserve Memory

**LSTM (Hochreiter & Schmidhuber, 1997)** adds a **cell state** $c_t$ that flows almost linearly through time. Instead of forcing everything into the hidden state, LSTM separates:

- $c_t$ — long-term memory
- $h_t$ — the exposed output/state for the current step

The cell state is controlled by learned **gates**. A gate is usually a vector in $[0,1]$, like a set of valves:

- *Forget gate* $f_t$ — how much old memory to keep
- *Input gate* $i_t$ — how much new information to write
- *Output gate* $o_t$ — how much to expose to the next step

The core update is:

$$
c_t = f_t \odot c_{t-1} + i_t \odot \tilde{c}_t
$$

This form matters because it gives memory an **additive** path through time. If the forget gate is near 1, the cell state can preserve old information across many steps without repeatedly multiplying by an arbitrary recurrent matrix.

<LSTMCellDiagram lang="en" />

<Note type="intuition">
LSTM gates and ResNet residual connections share the same insight: for a gradient to survive many steps, create a path where information mostly **adds** rather than repeatedly **multiplies**.
</Note>

**GRU** is a leaner variant: it merges some gates and does not keep a separate cell state $c_t$. It usually has fewer parameters than LSTM, trains much better than a vanilla RNN, and works well on many classical sequence tasks.

| | LSTM | GRU |
|---|---|---|
| Gates | forget, input, output | update, reset |
| State | cell $c_t$ + hidden $h_t$ | only $h_t$ |
| Parameters | more | fewer |

Both share the same core insight — gates create a near-additive update path to preserve memory longer — they only differ in how the gates are organized.

## Seq2Seq — Encoder-Decoder for Sequence to Sequence

Early RNN/LSTM/GRU models were often used for mapping one sequence to one label, or predicting step by step. But many tasks require mapping **one sequence to another sequence**:

- machine translation: English sentence → Vietnamese sentence
- speech recognition: audio frames → text
- summarization: long document → short summary

**Seq2Seq (Sutskever et al., 2014)** introduced the encoder-decoder framework:

1. The **encoder** reads the whole source sequence and produces a final state.
2. The **decoder** uses that state to generate the target sequence one token at a time.

The strength of encoder-decoder is that input and output can have different lengths. This was a major step: the model no longer merely classifies a sequence, it **translates** one sequence into another.

But classical Seq2Seq has a fatal weakness: the **fixed-vector bottleneck**. The encoder must squeeze the entire source sentence into one context vector. For long sentences, details near the beginning can disappear before the decoder needs them.

## Attention — Breaking the Bottleneck

The solution from Bahdanau et al. (2015) was to let the decoder "look back" at all encoder hidden states. Instead of receiving one fixed context vector, each decoder step builds its own context as a weighted sum over encoder states:

$$
c_t = \sum_{i=1}^{n} \alpha_{t,i} h_i
$$

Here $\alpha_{t,i}$ is the attention weight: at decoder step $t$, how much should the model attend to source position $i$?

<Seq2SeqAttentionDiagram lang="en" />

This is a crucial shift:

- Encoder states become a **memory bank**.
- The decoder state acts like a **query**.
- Attention retrieves the relevant memory at each output step.

<Note type="result">
Attention did not originally replace RNNs; it was attached to an RNN encoder-decoder to break the bottleneck. The Transformer later kept the idea of attention-as-memory, removed recurrence, and let every token attend directly to every other token.
</Note>

## The Remaining Problem: Sequential Is Slow

LSTM/GRU solves much of the memory problem; attention solves the Seq2Seq bottleneck. But the RNN family still has a system-level limitation: **temporal dependency**.

At step $t$, we need $h_{t-1}$. So the model cannot parallelize over the time axis the way a CNN processes a whole image or a Transformer processes a whole sequence. Training can still parallelize over batches and feature dimensions, but sequence length remains the hard axis.

This is why two major directions emerged after RNNs:

- **Drop recurrence and parallelize fully** → [Transformer & Attention](/dl-anatomy/architectures/transformer-en), where every token can attend to every other token. The cost: standard attention is $O(n^2)$ in sequence length.
- **Keep state but organize it as a structured linear scan** → [State Space Models](/dl-anatomy/architectures/state-space-models-en), where recurrence is designed to train/infer efficiently on long sequences.

<Note type="result">
The evolution here: RNN compresses the past into state → LSTM/GRU use gates to preserve memory longer → Seq2Seq turns RNNs into sequence-to-sequence models → attention turns encoder states into a memory bank. From there, the Transformer keeps attention and drops recurrence; SSMs return to state but make it scale linearly.
</Note>
